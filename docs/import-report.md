## Conflicts

### Conflict 1 — Surah 9 Ayah Count

| Attribute | Value |
|-----------|-------|
| Surah | 9 (التَّوۡبَةِ — At-Tawbah) |
| fahras.csv value | 128 |
| qurantexttanzil.csv value | 129 |
| Resolution | qurantexttanzil.csv used (Tier 1 source) |
| Explanation | A known scholarly variant. Some traditions count At-Tawbah as 128 ayahs; the Hafs An-Asim tradition (used by Tanzil) counts 129. The stored `ayah_count` for surah 9 is 129. |

---

## Assumptions

1. **fahras.csv surah number derivation.** The `col[0]` field in fahras.csv is an internal database ID, not the canonical surah number. The mapping `surah_number = col[0] - 226` was derived empirically: it produces the correct surah numbers for 113 of 114 surahs (verified against tanzil ayah counts). The only mismatch is surah 9 (At-Tawbah), which is a counting variant, not an ordering error.

2. **Surah names from quran.csv.** Surah names are sourced from the sura-header rows in quran.csv (rows where `aya_index = 0`). There are 224 such rows (not 114), because some surahs appear multiple times as headers. The first occurrence of each `surah_number` is used; duplicates are discarded.

3. **STEM token as canonical word form.** For the `words` table, the STEM morphological token is used as the representative form of each word. When a word has no STEM token (an edge case in the morphology corpus), the first token of that word is used instead. This occurred for 0 words in this build.

4. **Buckwalter → Arabic conversion.** All Arabic text in `*_arabic` columns is generated by converting the Buckwalter transliteration stored in `*_buckwalter` columns using the standard 38-character Buckwalter table. Characters not in the table are passed through unchanged.

5. **pages.csv col[0] interpretation.** The `col[0]` field in pages.csv contains values 1–604, identical to the row index. It is treated as a redundant page number, not as surah information. The `ayah_count_on_page` (col[1]) is the only value imported. Sum of all `ayah_count_on_page` values = 6,236, confirming correctness.

6. **quran.csv aya_index=0 rows excluded.** The quran.csv file contains 225 sura-header rows (aya_index=0) that are not canonical ayahs. These are used only for surah name extraction and excluded from the `ayahs` table.

7. **qurantexttanzil.csv trailing empty column.** Both qurantexttanzil.csv and quranuthmanitanzil.csv have a consistent empty 5th column (trailing comma artifact from export). This column is ignored during import.
