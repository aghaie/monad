# Q11 — Time Architecture Report (Phase ΩΣ)

**Method:** `foundational-questions-1.0` · corpus-only · deterministic.

## 1. Objective
How does the Quran model time — past/present/future, cyclic vs irreversible?

## 2. Method
Verb aspect as the temporal signal: PERF (completed/past), IMPF (ongoing/present-future), IMPV (command);
plus the FUT particle and T time-nouns. Recurrence (roots recurring in ≥10 ayahs) as a cyclic-vs-linear
proxy. Exact censuses.

## 3. Results
| signal | tokens | share |
|---|--:|--:|
| completed (PERF) | 9,150 | 47.3% |
| ongoing (IMPF) | 8,330 | 43.0% |
| command (IMPV) | 1,876 | |
| future particle (FUT) | 42 | |
| time nouns (T) | 1,166 | |
| roots recurring in ≥10 ayahs | 573 | |

## 4. Interpretation
Completed and ongoing aspect are **near-balanced** (47% vs 43%), with a heavy imperative load and pervasive
recurrence (573 roots recurring across the corpus). This is **not a linear chronology**: there is no
narrative arrow. Time is modelled **aspectually and recurrently** — completed events function as exemplars
re-invoked in a continual present-tense address, the same patterns recurring rather than advancing.

## 5. Falsification Attempts
Aspect counts are exact censuses. The recurrence figure is a direct document-frequency count. No inferential
claim is made beyond these.

## 6. Limitations
Aspect ≠ tense; Arabic aspect maps imperfectly to past/present/future. "Recurrent" is inferred from
repetition, not from an explicit cyclic marker.

## 7. Conclusion
**ASPECTUAL / RECURRENT, not linear-chronological.** The Quran balances completed-as-exemplar against
ongoing present-address, with heavy recurrence — time as pattern, not timeline.
