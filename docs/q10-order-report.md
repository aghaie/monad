# Q10 — Revelation Order vs Canonical Order Report (Phase ΩΣ)

**Method:** `foundational-questions-1.0` · corpus-only · deterministic.

## 1. Objective
Which ordering — canonical (mushaf) or revelation (chronological) — better explains the observed structure?

## 2. Method
**Honest blocker:** a full chronological revelation order (Nöldeke / Egyptian sequence) is **external data**
and therefore forbidden by the project's constraints. The corpus contains only the intrinsic **meccan /
medinan** partition. We compute and compare the frequency-residual structure of the two partitions.

## 3. Results
| partition | ayahs | residual |
|---|--:|--:|
| meccan | 4,593 | 0.807 |
| medinan | 1,621 | 0.821 |

The two partitions differ only marginally in residual structure (0.807 vs 0.821).

## 4. Interpretation
The canonical order is the **only intrinsic ordering** available; chronological order cannot be tested
without importing an external sequence. The meccan/medinan contrast — the one ordering signal the corpus
does carry — shows **near-identical residual structure**, i.e. the information-theoretic core does not depend
on that partition.

## 5. Falsification Attempts
A revelation-order null is **not constructible** without external chronology; this is stated rather than
faked. The meccan/medinan residuals are exact subgroup censuses.

## 6. Limitations
The headline comparison (canonical vs chronological) is unanswerable by construction. Only meccan/medinan is
available.

## 7. Conclusion
**UNKNOWN — chronological order is external and forbidden.** Within the corpus, meccan and medinan partitions
have near-identical residual structure; the canonical order is the only intrinsic order testable.
