# Q8 — Book vs Engine Report (Phase ΩΣ)

**Method:** `foundational-questions-1.0` · corpus-only · deterministic.

## 1. Objective
Is the Quran better modelled as a **knowledge repository** (book) or a **process generator** (engine)?

## 2. Method
Verb aspect as the static/process signal: PERF (completed/descriptive) = book; IMPF + IMPV (ongoing +
command) = engine. Plus COND conditionals and IMPV imperatives as explicit process triggers. Exact censuses.

## 3. Results
| signal | tokens |
|---|--:|
| static (PERF) | 9,150 |
| process (IMPF + IMPV) | 10,206 (**52.7%**) |
| imperatives (IMPV) | 1,876 |
| conditionals (COND) | 1,049 |

## 4. Interpretation
The text leans **engine**: process aspect outweighs completed aspect (52.7%), and it carries a heavy
imperative + conditional load (1,876 commands, 1,049 *if/then* structures). It is not a static repository of
facts but a register of commands and conditionals — a process-generating register applied to the addressee.

## 5. Falsification Attempts
Aspect counts are exact censuses. The conditional → choice link was independently the one robust decision
motif (Phase Δ), consistent with the engine reading.

## 6. Limitations
"Engine" here means a process/command register, not a claim about computation or semantics. Aspect is a
proxy for register.

## 7. Conclusion
**ENGINE-leaning.** Process aspect (52.7%) plus a heavy imperative/conditional load make the Quran more a
process generator than a static knowledge repository.
