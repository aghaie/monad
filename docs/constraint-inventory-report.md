# Constraint-Inventory Report — Phase Φ (A)

**Phase:** Φ — Counterfactual Quran Discovery Engine · **Method version:**
`counterfactual-discovery-1.0` · **Date:** 2026-06-08.
This phase measures **selection**, never truth, theology, revelation, or origin.

## 1. Objective
Collect the Phase-11+ surviving discoveries (strong/moderate only; weak forbidden) into the
constraint set C that defines "structurally-valid alternative Quran-like texts."

## 2. Method
Curated from the project record: frequency, consistency, motifs, dependency/co-occurrence, grammar,
locality, hub — each tagged with strength and its known status (e.g. hub = frequency; dependency real
in-sample but 0-generalizable per Phase P).

## 3. Results
Seven constraints (`constraint_inventory.json`). **C_frequency** (strong) is the dominant constraint
on lexical identity. **C_dependency / C_motif** (co-occurrence structure) are real (Phase 17, 3.9×
null) but 0-generalizable (Phase P). **C_consistency** is generic (Phase 15); **C_hub** = frequency
(Phase 16); **C_locality** is homogeneous (Phase 13/14).

## 4. Interpretation
The constraint set separates into two kinds: one that constrains **lexical identity** (frequency) and
ones that constrain **co-occurrence form** (dependency/motif). This distinction drives the whole
phase: structure constrains coherence; frequency weakly constrains identity.

## 5. Falsification Attempts
Weak findings are excluded by construction; each admitted constraint carries its prior falsification
status.

## 6. Limitations
Constraints are statistical (match distributions), not exact rules; the alternative space is the
max-entropy set matching them.

## 7. Conclusion
Seven robust constraints form C; they split into identity-constraints (frequency) and form-constraints
(co-occurrence). This is the basis for measuring the alternative space.
